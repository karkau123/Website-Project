    • 1. Introduction to OS
      • Definition & Functions Manages hardware, provides services for software. Functions: Resource Mgt, Process Mgt, Memory Mgt, I/O, Security.
      • Types of OS
        • Batch: Jobs processed in groups.
        • Time-Sharing: CPU time is shared among users.
        • Distributed: Manages a group of independent computers.
        • Real-Time (RTOS): Strict time constraints (Hard vs. Soft).
        • Embedded: Specialized for a specific device.
      • OS Structure
        • Monolithic: All services in kernel space. Fast but not robust.
        • Layered: OS divided into layers, each built on lower ones.
        • Microkernel: Minimal kernel, most services in user space.
        • Modular/Hybrid: Core kernel + loadable modules (e.g., Linux, macOS).
      • System Calls & Services Interface for programs to request OS services. Switches from User Mode to Kernel Mode.
    • 2. Process Management
      • Process vs Program: A program is passive code; a process is a program in execution.
      • Process States & PCB States: New, Ready, Running, Waiting, Terminated. PCB stores process info (state, PC, registers, etc.).
      • Scheduling
        • Schedulers: Long-term, Short-term, Medium-term.
        • Context Switch: Saving state of one process and loading another.
      • Scheduling Algorithms
        • FCFS: First-Come, First-Serve (Non-preemptive).
        • SJF: Shortest Job First (Preemptive/Non-preemptive). Optimal but needs future prediction.
        • Priority: Higher priority process runs first. Can lead to starvation.
        • Round Robin (RR): Time-slicing (quantum). Good for interactive systems.
      • Threads Lightweight process. User-level (fast, no kernel support) vs Kernel-level (slower, OS-managed).
      • Synchronization
        • Race Condition: Outcome depends on sequence of execution.
        • Critical Section: Code segment where shared resources are accessed.
        • Mechanisms
          • Mutex: Mutual Exclusion lock. Only one thread can own it.
          • Semaphore: Counting signal. Allows N threads to access.
          • Monitor: High-level construct with mutual exclusion and condition variables.
          • Condition Variables: Allow threads to wait for a condition to become true.
    • 3. CPU Scheduling
      • Criteria CPU utilization, Throughput, Turnaround Time, Waiting Time, Response Time.
      • Preemptive vs Non-preemptive Preemptive can interrupt a running process; Non-preemptive runs until completion/block.
    • 4. Deadlocks
      • Characterization Mutual Exclusion, Hold and Wait, No Preemption, Circular Wait. All four must hold.
      • Resource Allocation Graph: Visualizes processes and resources. Cycle implies deadlock.
      • Handling
        • Prevention: Negate one of the four conditions.
        • Avoidance: Use algorithms like Banker's to ensure a safe state.
        • Detection & Recovery: Allow deadlocks, detect them, then recover (e.g., terminate process).
    • 5. Memory Management
      • Logical vs Physical Address: CPU sees logical, MMU translates to physical.
      • Techniques
        • Contiguous: Process occupies a single block. Leads to external fragmentation.
        • Paging: Memory divided into fixed-size frames, process into pages. Solves external fragmentation.
        • Segmentation: Memory divided into logical segments (code, data, stack).
      • Virtual Memory
        • Demand Paging: Load pages into memory only when needed.
        • Page Replacement Algos
          • FIFO: First-In, First-Out.
          • LRU: Least Recently Used.
          • Optimal: Replace page that won't be used for longest time.
        • Thrashing: High paging activity, low CPU utilization.
      • TLB (Translation Lookaside Buffer): A fast cache for page table entries.
    • 6. File System Mgt
      • File Concepts: Attributes (name, size, date), Operations (create, read, write).
      • Access Methods: Sequential, Direct, Indexed.
      • Directory Structure Single-Level, Two-Level, Tree-Structured, Acyclic-Graph.
      • Allocation Methods
        • Contiguous: Simple, fast, but suffers from fragmentation.
        • Linked: Blocks are scattered, linked by pointers. Solves fragmentation.
        • Indexed: Uses an index block to store pointers to data blocks.
      • Free Space Management: Bit Vector, Linked List, Grouping.
      • Journaling: Logs changes before writing to ensure consistency after a crash.
    • 7. I/O Systems
      • I/O Hardware: Devices, Controllers, Buses.
      • I/O Techniques
        • Programmed I/O: CPU polls device status.
        • Interrupt-driven I/O: Device interrupts CPU when ready.
        • DMA (Direct Memory Access): Controller transfers data directly to/from memory.
      • Disk Scheduling
        • FCFS, SSTF, SCAN, C-SCAN, LOOK.
      • RAID Levels RAID 0 (Stripe), RAID 1 (Mirror), RAID 5 (Parity), RAID 10 (Stripe of Mirrors).
    • 8. Storage Management
      • Secondary Storage: Non-volatile memory (HDD, SSD). Managed by file system.
      • Backup Strategies Full, Incremental, Differential. 3-2-1 Rule (3 copies, 2 media, 1 offsite).
    • 9. Security & Protection
      • Goals: Confidentiality, Integrity, Availability (CIA Triad). Principle of Least Privilege.
      • Authentication: Passwords, MFA, Biometrics, Tokens.
      • Threats: Malware (Virus, Worm, Ransomware), Phishing, DoS.
      • Mechanisms: Firewalls, Antivirus, Encryption (Symmetric & Asymmetric).
    • 10. System Calls & Kernel
      • User vs Kernel Mode Dual-mode operation for protection. Kernel mode has privileged access. Transition via system call.
      • Examples (Linux/Windows) fork()/CreateProcess, read()/ReadFile, write()/WriteFile, open()/CreateFile.
    • 11. Distributed Systems
      • Concepts: Manages multiple independent CPUs. Goal is transparency.
      • RPC (Remote Procedure Call): Allows a program to execute a procedure on another computer.
      • Distributed File Systems (DFS): e.g., NFS, GFS.
      • Clock Synchronization: Lamport Timestamps, Vector Clocks.
    • 12. Virtualization & Cloud
      • Virtualization Creates a virtual version of a resource (e.g., server, storage).
      • Hypervisors
        • Type 1 (Bare-Metal): Runs directly on host hardware (e.g., ESXi, KVM).
        • Type 2 (Hosted): Runs on a conventional OS (e.g., VirtualBox).
      • Containers vs VMs VMs virtualize hardware. Containers virtualize the OS (share kernel). Lighter, faster.
      • Cloud OS Manages resources in a cloud environment (e.g., OpenStack, Kubernetes for orchestration).
